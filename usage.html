1. About

ScreenCap is a desktop recording program that allows you to record your desktop or another window.

2. Usage

To record choose the settings and click Record. When you are done, click Stop.

3. Settings

3.1. Window

Click on Window to select a window. When the Window button is pushed, click in the Window you want to record. To select the desktop window, click in the titlebar of any window. When you select a window, ScreenCap automatically centers it to make shure the entire window is visible.

3.2. Crop rect

Select the width and height of output video. Note that the rectangle is always centered.

3.3. Output file

Click on output file to select output file. The opened dialog box should be familiar.

3.4. Frame rate

The frame rate sets the screenshot rate. When Benchmark is selected, this setting has no effect.

3.5. Benchmark

Activate Benchmark to measure the maximum frame rate the system can perform using the compression and system usage settings.

3.6. Compression method

The compression method determines the algorithm chosen by zlib. These are probably better documented by the zlib project.

3.6.1. Z_DEFAULT_STRATEGY

This algorithm gives good compression at expence of CPU usage

3.6.2. Z_FILTERED

Each frame produced by ScreenCap is actually encoded using the PNG difference filter. Select Z_FILTERED if you want zlib take advantage of that.

3.6.3. Z_RLE

Force zlib to use run-length encoding. This is cheaper than Z_DEFAULT_STRATEGY.

3.6.4. Z_HUFFMAN_ONLY

This option forces zlib to not use any string comparison. Using only Huffman encoding is even cheaper than Z_RLE.

3.6.5. Z_FIXED

Force zlib to use a fixed block size.

3.7. Compression level

Select 1 for a large file and small CPU usage and 9 for the opposite.

3.8. #Threads

This setting sets the upper limit of how many encoding threads that the program will run. The highest value corresponds to the number of logical processors the machine has.

3.9. Process priority

This setting tells the scheduler in the operating system how much time it should give ScreenCap. Beginning with the process with highest priority, the order is

  1. REALTIME_PRIORITY_CLASS
  2. HIGH_PRIORITY_CLASS
  3. ABOVE_NORMAL_PRIORITY_CLASS
  4. NORMAL_PRIORITY_CLASS
  5. BELOW_NORMAL_PRIORITY_CLASS
  6. IDLE_PRIORITY_CLASS

4. Bugs and other issues

4.1. Window selection

The window selection seems a bit confusing. What does WindowFromPoint really return?

4.2. Output file

Currently, the program just dumps a huge list of png files prefixed by given output filename. The audio track will also use this prefix followed by "-audio.wav". In a future release, the individual frames should be embedded into a MNG-VLC stream. I have not found an example file though so I cannot test if avconv takes an MNG.

4.3. Audio track

For now the audio track is just a CD quality RIFF-WAVE file. It should be possible to choose sample rate, bit depth and number of channels. Also the input is currently just WAVE_MAPPER. The MIDI_MAPPER has disappeared in recent versions of Microsoft Windows. What will happen to WAVE_MAPPER?

4.4 Compilation of the program

ScreenCap was build using Code::Blocks and MinGW. If you do not want to use Code::Blocks, just compile all cpp-files and link them together with zlib and the nessecary Windows libraries: gdi32 user32 and winmm.
